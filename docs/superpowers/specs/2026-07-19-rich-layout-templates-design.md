# ObsidianToSM 丰富排版模板设计

## 目标

在保留现有六套颜色主题的基础上，新增四套结构明显不同的公众号排版模板。排版模板和颜色主题可以自由组合，右侧预览、复制富文本、创建草稿和直接发表必须使用完全相同的 HTML。

## 已确认范围

- 保留六套颜色主题：简约黑白、科技蓝、商务绿、人文墨、杂志红、暖橙生活。
- 新增四套排版模板：杂志叙事、现代商务、阅读手记、技术蓝图。
- 排版模板增加“不套用模板”选项。
- “不套用模板”保持当前基础排版，只应用颜色主题。
- 排版模板和颜色主题相互独立，可以任意组合。
- 记住用户最后选择的排版模板和颜色主题，关闭并重新打开 Obsidian 后仍然有效。
- 不增加在线模板市场、模板导入和自定义模板编辑器。

## 交互设计

右侧工作台的操作按钮维持现有顺序。颜色色块从操作按钮组中移到独立的“样式栏”，排版模板下拉框和六个颜色色块放在同一行：

1. 左侧为排版模板下拉框，选项依次为“不套用模板”“杂志叙事”“现代商务”“阅读手记”“技术蓝图”。
2. 右侧保留六个颜色色块，当前颜色使用清晰描边标记。
3. 侧栏宽度不足时，样式栏允许自然换行，不压缩操作按钮和预览区域。
4. 切换排版模板或颜色后立即刷新当前笔记预览。
5. 下拉框和色块均提供中文无障碍标签与悬停提示。

## 组合规则

排版模板负责元素结构和视觉层级，颜色主题负责强调色、标题色、正文色、引用背景和代码背景。

- 排版模板选择“不套用模板”时，沿用现有标题、引用、代码和图片样式。
- 选择四套新模板时，模板读取当前颜色主题，并将强调色应用到标题装饰、边框、链接、列表标记和表格表头等位置。
- 模板可使用中性色作为纸张背景、辅助线和代码前景色，但不得写死主要强调色。
- 用户自定义 CSS 仍在内置样式之后输出，保留最终覆盖能力。

## 四套模板

### 杂志叙事

- 适合随笔、人文、人物故事和长篇叙事。
- 一级标题居中，使用较强的封面式层级和短分隔线。
- 二级标题使用衬线字体和章节感留白。
- 引用采用整块浅色底，不使用默认左边框。
- 图片留出更大上下间距，正文采用舒展行高。

### 现代商务

- 适合方法论、行业分析、工作复盘和观点文章。
- 一级标题左对齐，标题下方使用强调色边线。
- 二级标题采用强调色侧边框或底色标签。
- 引用使用结论卡片样式，列表和表格突出信息扫描效率。
- 表头、链接和关键分隔线统一使用当前颜色主题的强调色。

### 阅读手记

- 适合读书笔记、生活记录、书评和影评。
- 使用温和的纸张背景与衬线字体组合，但正文保持清晰可读。
- 二级标题采用书签式侧边装饰。
- 引用表现为摘录卡片，图片和段落间距偏舒展。
- 列表标记和小标题装饰使用当前颜色主题的强调色。

### 技术蓝图

- 适合教程、工具测评、开发记录和知识说明。
- 标题使用清晰的编号感、分隔线和紧凑层级。
- 提示引用采用信息块样式。
- 代码块使用深色高对比底色，保留语法高亮和可选行号。
- 行内代码、表格、链接和列表强调技术文档的可扫描性。

## 渲染设计

新增独立的排版模板定义，不把模板样式继续堆入颜色主题对象。渲染输入由单一 `themeId` 扩展为：

```ts
interface RenderOptions {
  themeId?: string;
  layoutId?: string;
  customCss: string;
  enableLineNumbers: boolean;
}
```

每个模板为公众号常用元素提供样式或标记生成规则，覆盖：

- 文章容器
- 一级、二级、三级标题
- 段落、粗体、斜体和链接
- 引用、无序列表、有序列表和列表项
- 分隔线
- 图片
- 行内代码和代码块
- 表格、表头和单元格

关键样式必须直接写入元素的 `style` 属性。不得依赖伪元素、复杂选择器、外部字体和脚本，因为这些内容可能被微信公众号编辑器或 API 过滤。末尾仍保留内置 CSS 和用户自定义 CSS，供 Obsidian 预览及兼容性兜底。

渲染器只生成一份最终 HTML；预览、复制、草稿和发表都调用同一个准备笔记流程，不单独维护第二套模板逻辑。

## 状态与兼容

- 设置新增 `layoutId`，默认值为 `none`。
- 现有 `themeId` 继续使用，默认值保持商务绿。
- 读取旧版本设置时缺少 `layoutId`，自动使用 `none`，旧用户升级后文章外观不会突然改变。
- 未知的 `layoutId` 回退到 `none`；未知的 `themeId` 继续回退到简约黑白。
- 工作台控制器同时保存 `themeId` 和 `layoutId`，任何一个变化都重新渲染预览。
- 每次选择变化后写入插件设置，使选择跨会话保留。

## 测试与验收

### 自动化测试

- 四套模板和“不套用模板”的 ID、中文名称及回退规则正确。
- 现有六套颜色主题及回退规则保持不变。
- 同一篇 Markdown 在四套模板下输出不同的标题、引用、列表、代码和表格内联样式。
- 切换颜色主题后，四套模板中的强调色随之变化。
- “不套用模板”保持当前基础排版输出。
- 控制器切换模板或颜色时均会重新加载当前笔记。
- 缺少新设置字段的旧数据能使用默认模板正常加载。

### 手工验收

- 右侧工作台的样式栏紧凑显示模板下拉框和六个色块。
- 模板与颜色任意组合时，长文预览可正常滚动，无控件重叠。
- 标题、正文、引用、列表、表格、图片、行内代码和代码块均有完整样式。
- 复制到公众号编辑器后的排版与右侧预览基本一致。
- 创建草稿和直接发表获得的排版与预览一致。
- 重启 Obsidian 后恢复上次选择。

## 非目标

- 不复制 NoteToMP 或其他闭源产品的模板代码与素材。
- 不引入远程模板服务或第三方运行时依赖。
- 不改变公众号账号、封面、草稿和发表 API 流程。
- 不在本次改动中重构与主题无关的插件代码。

## 评审调整：左右预览工作区

根据实际安装后的界面反馈，工作台由连续横向区域改为响应式左右布局：

- 左侧固定为封面与操作菜单，依次保留刷新、复制、草稿、发布和帮助。
- 右侧从上至下为公众号选择、六个颜色主题、排版模板选择、预览模式和文章预览。
- 预览模式使用手机与 PC 两个分段图标按钮。手机模式将正文限制为约 `375px` 并居中，PC 模式使用右侧可用宽度。
- 默认使用 PC 预览，并持久化最后一次预览模式选择。
- 工作台宽度小于 `720px` 时自动改为上下布局，避免封面、控件和正文互相挤压。
- 帮助内容显示在左侧菜单下方，不占用右侧预览空间。

## 二次评审调整：封面与菜单并排

实际目标不是把工作台长期分成左右两栏，而是只让顶部信息区并排：

- 顶部左侧只显示封面。
- 顶部右侧从上至下显示公众号、颜色主题、排版模板和预览模式。
- 刷新、复制、草稿、发布和帮助五个菜单按钮位于顶部右侧区域的底部，并靠右排列。
- 完整文章预览位于顶部信息区下方，使用工作台完整宽度并独立滚动。
- 面板宽度不足时，顶部信息区自动改为封面在上、设置和菜单在下，文章预览仍位于最后。
