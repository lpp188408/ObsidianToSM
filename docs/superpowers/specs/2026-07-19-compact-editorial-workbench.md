# ObsidianToSM 紧凑编辑部工作台设计

## 目标

将右侧发布工作台改为紧凑、有层次的编辑部工具台。操作区占用尽量少的垂直空间，让文章预览成为视觉主体。

## 布局

- 顶部为 `ObsidianToSM` 状态行，显示当前预览状态，不使用卡片容器。
- 下一行是 `48×64px` 的封面缩略图、账号下拉和打开公众号后台的外链图标。
- 操作区为单行工具栏：刷新、复制、草稿、发布、主题、帮助。
- 发布按钮是唯一高强调按钮；草稿按钮使用中等强调；其余使用无文字图标按钮，并提供中文 tooltip。
- 主题使用六个色块。点击色块立即刷新预览；鼠标悬停显示主题名称。
- 帮助展开为紧凑的内联提示条，不遮挡预览。
- 文章区紧接工具栏下方，使用细分隔线和可滚动预览，不再出现大封面卡片或大面积空白。

## 视觉规范

- 使用 Obsidian 原生 `--background-*`、`--text-*` 和 `--interactive-*` CSS 变量。
- 仅发布按钮使用当前主题的强调色；不使用渐变、厚阴影和大圆角。
- 工具按钮固定 `30×30px`，封面缩略图固定 `48×64px`，避免内容变化导致布局跳动。
- 未设置封面时显示图片加号图标与虚线边框；已设置封面时显示裁剪后的缩略图。
- 图标使用 Obsidian 内置 Lucide 图标，按钮的可访问标签使用中文。

## 验收

- 右侧栏顶部操作区在常见宽度下不超过约 118px 高度。
- 未设置账号和封面时预览仍可用。
- 所有原有操作仍保留，发布和草稿在视觉上明确区分。
- 主题切换、账号选择、复制、刷新、草稿和发布行为不变。
